# v12.0.0 roadmap — bash removed entirely

> **Status at v11.2.0 (76f8afc):** `legacy/scripts/` is canonical. The `scripts/` symlink is removed. Operator integrations have been updated. v12.0.0 is the long-term horizon — delete `legacy/scripts/` entirely; Go is the only runtime.
>
> **Estimated effort:** 6+ months. Recommended as a series of sub-releases (v11.3.0 → v11.7.0 → v12.0.0), NOT a single change-window.

## Hard blockers

These must all be resolved before `rm -rf legacy/` is safe.

### 1. Native Go ship implementation

**Current state:** `go/internal/phases/ship/ship.go:defaultShipScript = "legacy/scripts/lifecycle/ship.sh"`. Go shells out for atomic commit+push.

**What native Go ship must do** (matches the 23-test ship-integration-test matrix):
- `git diff HEAD` for tree-state SHA (audit-binding model)
- Three commit classes:
  - `cycle` — full audit-binding (recent PASS, SHA match, HEAD/tree bound, `acs-verdict.json:red_count == 0`)
  - `manual` — interactive y/N confirmation (CI mode: `EVOLVE_SHIP_AUTO_CONFIRM=1`)
  - `release` — pipeline-internal, skips audit (version-bump.sh mutates post-audit)
- SHA-pinning via `.evolve/state.json:expected_ship_sha` + `expected_ship_version` (TOFU within plugin version)
- Atomic `git commit` + `git push`
- `gh release create` for release class
- Challenge-token + tamper-evident ledger entry (`prev_hash` + `entry_seq`)
- Honor env vars: `EVOLVE_BYPASS_SHIP_VERIFY`, `EVOLVE_SHIP_AUTO_CONFIRM`, `EVOLVE_BYPASS_PREFIX_GATE`

**Test parity gate:** `legacy/scripts/tests/ship-integration-test.sh` has 23 tests. Add `go/internal/phases/ship/ship_native_test.go` with the same test matrix against the native impl. Both must pass.

**Sub-release: v11.3.0.**

### 2. Native Go guards (hooks)

**Current state:** `.claude/settings.json:PreToolUse` hooks call `bash $CLAUDE_PROJECT_DIR/legacy/scripts/guards/{ship-gate,phase-gate-precondition,role-gate}.sh`. Plus `legacy/scripts/hooks/{research-quota-gate,doc-deletion-guard}.sh`.

**What native Go guards need:**
- All 5 hooks reimplemented as Go subcommands (already partially done in `go/internal/guards/`)
- JSON-on-stdin contract preserved (PreToolUse hook input format)
- Exit codes: 0=allow, 2=deny (the Claude Code hook protocol)
- `.evolve/guards.log` append format unchanged

**Migration:**
- Update `.claude/settings.json` to invoke `bash $CLAUDE_PROJECT_DIR/go/bin/evolve guard <name>` (or, when the binary is on PATH, just `evolve guard <name>`)
- Provide a backcompat shim period: keep bash versions in `legacy/scripts/guards/` for 1-2 sub-releases
- Then delete

**Sub-release: v11.4.0.**

### 3. EGPS predicate ports

**Current state:** `acs/regression-suite/cycle-*/*.sh` has 196 bash predicates. `go/acs/cycle*/predicates_test.go` has 120/272 Go counterparts.

**Effort estimate:** the remaining 152 predicates (across cycles 44-141) follow the same patterns as cycle43. Mechanical port. ~2 days of focused work.

**Sub-release: v11.5.0.** Delete `acs/regression-suite/` after Go side is complete.

### 4. Native Go dispatch

**Current state:** `legacy/scripts/dispatch/run-cycle.sh` + `legacy/scripts/dispatch/evolve-loop-dispatch.sh` are the canonical cycle drivers. Go side has `evolve cycle run` + `evolve loop` but bash versions still authoritative for some operator flows.

**What needs to happen:**
- Audit every operator flow that uses `bash run-cycle.sh` directly. Most are documented in CLAUDE.md / AGENTS.md as preferred entry points.
- Switch operator-facing docs to `evolve cycle run` / `evolve loop`
- Keep bash versions as backcompat shims (call `evolve cycle run` internally) for one sub-release
- Delete

**Sub-release: v11.6.0.**

### 5. Release pipeline

**Current state:** `legacy/scripts/release-pipeline.sh` is the canonical release driver. It calls `release/preflight.sh`, `release/changelog-gen.sh`, `release/version-bump.sh`, `utility/release.sh`, `lifecycle/ship.sh`, `release/marketplace-poll.sh`.

**Option A** — port to `evolve release X.Y.Z`. Complex. The pipeline does git pulls in `~/.claude/plugins/marketplaces/`, version-bumps across multiple files, polls marketplace propagation. Substantial Go reimplementation.

**Option B** — document `release-pipeline.sh` as the one bash exception. v12.0.0 removes most of `legacy/scripts/` but keeps `legacy/scripts/release-pipeline.sh` + its helpers (~10 files) under a `tools/` directory. Pragmatic.

**Recommendation:** Option B. Releases are infrequent enough that the cost-of-port doesn't justify the gain.

**Sub-release: v11.7.0** (if Option A) or **skipped** (if Option B).

### 6. resolve-roots.sh consumers

**Current state:** Many bash scripts source `legacy/scripts/lifecycle/resolve-roots.sh` for `EVOLVE_PLUGIN_ROOT` / `EVOLVE_PROJECT_ROOT` resolution.

**Resolution:** Once #1-#5 are done, the only bash left is what's in tools/ (if Option B above). Those can keep sourcing resolve-roots.sh from a `tools/` path.

## Gates between sub-releases

Each sub-release must pass:
1. All 4 preflight gate-test suites (`bash legacy/scripts/tests/{guards,ship-integration,role-gate,phase-gate-precondition}-test.sh`)
2. Full Go regression (`cd go && go test -race -count=1 ./...`)
3. One real cycle through `evolve cycle run` (paid; budget cap $5)
4. Parity audit (`bash legacy/scripts/parity-audit.sh --simulate` for no-LLM gate; `--full` for paid full gate)

## Don't-rediscover list

These were learned during v11.0.0 → v11.2.0 and shouldn't bite v12.x:

- `EVOLVE_SHIP_AUTO_CONFIRM=1` breaks ship-integration-test in preflight — never set when invoking `release-pipeline.sh`
- README `Current (vMAJOR.MINOR)` form is mandatory — patch suffix breaks `release.sh` regex
- Marketplace checkout accumulates dirty profiles — `git -C ~/.claude/plugins/marketplaces/evolve-loop stash` before each release
- `BASH_SOURCE[0]/../..` path arithmetic is fragile — use `resolve-roots.sh`-style walk-up
- Perl replacement uses `(?<!legacy/)pattern/` negative lookbehind to avoid double-prefixing
- `_test.go` filename suffix excludes from `go build` — use `_testbuild.go` for build-tag-gated test fixtures
- Blanket perl creates doc rot — manual cleanup of migration tables / deprecation schedules required after
