# v11.2.0 roadmap (now SHIPPED) + v12.0.0 horizon

> **Status:** v11.0.0 + v11.1.0 + v11.2.0 all SHIPPED. The bash `scripts/` symlink has been removed. Every in-repo reference uses `legacy/scripts/...` directly. v12.0.0 (bash removed entirely) remains the long-term horizon.

## What v11.2.0 did

Removed the backcompat `scripts/ → legacy/scripts/` symlink. Every reference in:
- Internal bash-to-bash refs (~196 `legacy/scripts/**/*.sh` files)
- Agents + skills markdown (~53 files)
- Docs (~82 files)
- Plugin manifests + hook commands (`.claude-plugin/`, `.claude/settings.json`)
- Top-level `CLAUDE.md`, `AGENTS.md`, `README.md`, `GEMINI.md`
- Go source defaults (`ship.go`, etc.) + Go ACS predicates

...now point at `legacy/scripts/...` directly. The hardcoded `BASH_SOURCE[0]/../..` path arithmetic in 196 bash scripts was updated to `../../..` to account for the extra `legacy/` depth.

`resolve-roots.sh` got a walk-up fallback that finds the repo root by looking for `.claude-plugin/` rather than hardcoding depth.

## Breaking change

Operator hooks pointing at `$CLAUDE_PROJECT_DIR/scripts/...` MUST be updated to `$CLAUDE_PROJECT_DIR/legacy/scripts/...`. There is no symlink fallback as of v11.2.0.

## Lessons from v11.2.0 execution

1. **Path arithmetic via `BASH_SOURCE[0]/../..` is fragile under directory restructuring.** 98 scripts had this hardcoded. The fix needed both depth update AND a `pwd -P` consideration for symlink invocation paths.
2. **Symlink invocation breaks `cd ... && pwd` arithmetic.** `pwd` without `-P` returns the *logical* path the caller used — which doesn't include the resolved symlink target. So `../..` from a symlinked path lands somewhere different than `../..` from the real path. The fix: invoke directly via the real path (`legacy/scripts/...`), not via the symlink.
3. **`resolve-roots.sh` should not hardcode depth.** The walk-up-to-`.claude-plugin/` pattern is more resilient and survives any future restructuring.
4. **Perl `(?<!legacy/)scripts/` is the safe replacement pattern.** Negative lookbehind avoids `legacy/legacy/` double-prefix bugs.
5. **Doc rot from blanket perl is real.** The deprecation table in `docs/migration-from-bash.md` had self-referential prose like "scripts/ is a symlink to legacy/scripts/" that became gibberish after `scripts/` → `legacy/scripts/`. Manual cleanup needed.
6. **ACS Go predicates use `filepath.Join(root, "scripts", ...)`** — separate string args, not slash-suffixed. Needed a different perl: `, "scripts", → , "legacy", "scripts",`.

## v12.0.0 — remains the long-term horizon

**Goal:** delete `legacy/scripts/` entirely. Go is the only runtime.

**Blockers (unchanged from v11.0.0 planning):**

1. **Go shells out to bash for `ship.sh`** — needs native Go ship implementation (git diff HEAD, audit-binding, atomic commit+push, gh release create, challenge-token ledger).
2. **Other shell-outs** in observer, sandbox, profile-validation paths.
3. **EGPS predicates are bash** under `acs/regression-suite/cycle-*/*.sh` — need Go counterparts.
4. **Hook system** — `.claude/settings.json` hooks invoke bash scripts. Either hooks become Go binary invocations OR bash stays for hooks only.

**Estimated effort:** 6+ months. Not a near-term release.
