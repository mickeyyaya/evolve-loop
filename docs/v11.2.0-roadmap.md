# v11.2.0 + v12.0.0 roadmap (post-v11.1.0)

> **Status:** v11.0.0 + v11.1.0 SHIPPED. v11.2.0 + v12.0.0 deferred — they require multi-session work beyond what's safe to cram into a single change-window. This doc captures what's left so a future session can pick it up cleanly.

## Where we are at v11.1.0 (b668fc2)

- Go binary is `tier-1-primary` in plugin manifest, optional accelerator with `EVOLVE_USE_LEGACY_BASH=1` rollback
- Bash scripts physically at `legacy/scripts/`; `legacy/scripts/` is a backcompat symlink → `legacy/scripts/`
- All in-repo `legacy/scripts/...` references continue to resolve through the symlink (CLAUDE.md, AGENTS.md, agents/, skills/, hooks/, Go source defaults, internal bash-to-bash refs — everything)
- Parity-audit harness (`legacy/scripts/parity-audit.sh`) supports `--dry-run`, `--simulate`, `--full` modes
- Perf comparison script (`legacy/scripts/perf-cycle-comparison.sh`) ready for operator-driven runs

## v11.2.0 — what it would take

**Goal:** remove the `legacy/scripts/` symlink. Every reference must use `legacy/scripts/...` directly.

**Scope (~245 files to update):**

| Surface | Count | Pattern |
|---|---|---|
| Internal bash-to-bash references | ~110 | `legacy/scripts/**/*.sh` referencing each other via `legacy/scripts/...` |
| Agents + skills markdown | ~53 | `agents/*.md`, `skills/*/SKILL.md` referencing `legacy/scripts/...` |
| Docs | ~82 | `docs/**/*.md` referencing `legacy/scripts/...` |
| Plugin manifests | 5+ | `.claude-plugin/plugin.json`, `marketplace.json`, `.claude-plugin/commands/*.md` |
| Top-level docs | 5+ | `CLAUDE.md`, `AGENTS.md`, `README.md`, `CHANGELOG.md`, `GEMINI.md` |
| Go source defaults | ~10 | `go/cmd/evolve/cmd_*.go`, `go/internal/phases/ship/ship.go` (defaultShipScript), etc. |
| Test scripts | many | `legacy/scripts/tests/*.sh` (also at `legacy/scripts/tests/`) |

**Migration approach (recommended):**

1. **Phase A — direct paths in Go source** (low risk, isolated):
   - Update `defaultShipScript = "legacy/scripts/lifecycle/ship.sh"` → `"legacy/scripts/lifecycle/ship.sh"` in Go.
   - Update similar defaults in `cmd_guard.go`, `cmd_doctor.go`, `cmd_worktree.go`, `failureadapter.go`, etc.
   - Update tests that assert these defaults.
   - Verify Go tests green.
   - Ship as one commit.

2. **Phase B — bash-to-bash internal refs** (highest internal risk):
   - For each `legacy/scripts/**/*.sh`, sed-replace `legacy/scripts/` → `legacy/scripts/` ONLY where it's a path token (not inside prose comments).
   - Best done with a careful `grep -l 'legacy/scripts/' legacy/scripts/**/*.sh | xargs sed -i ...` plus per-file review.
   - Run all 4 preflight gate-test suites after.
   - Ship as one commit.

3. **Phase C — manifests + hooks** (operator-facing):
   - Update `.claude-plugin/plugin.json`, `marketplace.json`, hook commands in `.claude/settings.json`.
   - Test: `bash legacy/scripts/lifecycle/ship.sh --help` still works (via symlink) AND `bash legacy/scripts/lifecycle/ship.sh --help` works.
   - Ship as one commit.

4. **Phase D — docs + agents + skills** (low runtime risk, high reader confusion if missed):
   - sed across agents/, skills/, docs/.
   - Be careful with prose: "Bash scripts:" should stay, "bash legacy/scripts/lifecycle/ship.sh" must move.
   - Ship as one commit.

5. **Phase E — remove the symlink** (final breaking change):
   - `rm scripts && git add -A`
   - Verify full pipeline still works end-to-end (`bash legacy/scripts/...` should now fail; only `bash legacy/scripts/...` should work).
   - Ship as one commit.

6. **Phase F — v11.2.0 release**:
   - Update `docs/migration-from-bash.md` deprecation timeline.
   - Update README "Current (v11.2)".
   - Run `legacy/scripts/release-pipeline.sh 11.2.0`.

**Estimated effort:** 4-8 hours of focused work + the release pipeline. Recommended as a dedicated session.

**Breaking-change communication:**
- Bump major-minor (v11.1 → v11.2)
- Auto-changelog generates the entry
- README + migration doc explicitly call out: "Operator hooks pointing at `$CLAUDE_PROJECT_DIR/legacy/scripts/...` MUST be updated to `$CLAUDE_PROJECT_DIR/legacy/scripts/...`."

## v12.0.0 — bash removed entirely

**Goal:** delete `legacy/scripts/` entirely. Go is the only runtime.

**Blockers:**

1. **Go shells out to bash for `ship.sh`** — per Phase 2 progress doc: "Ship phase uses CmdRunner injection — shells to legacy/scripts/lifecycle/ship.sh." Need a native Go ship implementation that does:
   - `git diff HEAD` for tree-state SHA
   - audit-binding verification
   - atomic git commit + push
   - `gh release create` for release class
   - challenge-token ledger entry
2. **Other shell-outs** — likely in observer, sandbox, profile-validation paths. Audit needed.
3. **EGPS predicates are bash** — `acs/regression-suite/cycle-*/*.sh` would need Go counterparts OR a separate runner that survives bash removal. Currently both bash and Go ACS suites coexist; v12.0.0 would either remove bash entirely (lose those predicates) or keep just the EGPS subset.
4. **Hook system** — `.claude/settings.json` hooks invoke bash scripts. Either the hooks become Go binary invocations OR bash stays for hooks only (contradicting "bash removed entirely").

**Estimated effort:** 6+ months. Multiple cycles per blocker.

**Recommended approach:** treat v12.0.0 as the long-term horizon, not a near-term release. v11.x line can continue for 12+ months without it.

## Other Phase 4 deferrals (not version-gated)

- **Plugin loader `binaries[]` integration** — BLOCKED externally. Claude Code plugin spec doesn't formally consume the `binaries[]` field we added in v11.0.0. The field is forward-looking metadata; it activates whenever the spec adopts it. No action on our side until then.
- **Real-cycle perf comparison numbers** — script exists at `legacy/scripts/perf-cycle-comparison.sh`. Operator runs it manually (~$5-15 per side) and pastes the report into a github discussion. Not a release-blocking item.
