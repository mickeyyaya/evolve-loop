Phase pipeline

Phases are configuration; legality, spine, and the ship floor are kernel-validated code.

How it works today

  • The registry is the SSOT (docs/architecture/phase-registry.json, schema v4): 15 phases — intent, scout, triage, spec-verify, architecture-design, plan-review, tdd, build-planner, build, doc-sync, tester, audit, ship, retrospective, memo. Spine order: intent → scout → triage → tdd → build-planner → build → audit → ship. Goal recipes insert conditional phases (coverage-gate, adversarial-review, scans…) per goal type; the single-word phase vocabulary is CLOSED (micro-phase-catalog.md).
  • Orchestrator = bounded loop (go/internal/core/orchestrator.go:659-865): plan → select → dispatch → review → record, with a chokepoint-escape guard (ADR-0044) so no cycle can exit unrecorded.
  • State machine legality (statemachine.go:170-203): explicit graph — audit→{ship,retro}, retro→{ship,tdd,end}, etc. Verdict branches resolve from each phase spec’s on_pass/on_fail.
  • Spine gate (SpineSatisfiedUpTo, statemachine.go:368-386): a mandatory anchor runs only if every earlier mandatory anchor left a real on-disk handoff artifact. Currently fail-open (76 warnings in one batch — queued: spine-failopen-telemetry).
  • The floor is not config: router.DefaultShipFloor() = {tdd, build, audit} (floor.go:55) — ship implies those ran, policy can relax only build/tdd (trivial class), never audit.
  • Contract gate: every phase’s deliverable is checked for well-formedness (path, sections, parseable verdict), default enforce (ADR-0034), 2 correction re-dispatches, circuit breaker after repeated blocks.
Model proposes, kernel disposes — the LLM advisor proposes per-cycle phase plans and per-phase {cli, tier}; the router clamps every proposal against mandatory phases, insertion caps (registry default 6), profile envelopes, and the ship floor. Clamps are recorded (Clamp{Rule,Proposed,Forced}), never silent.
